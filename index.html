Workflow-guide

Tilføj 10 nye kampe (ny runde)
1
Åbn notebooken i Google Colab. Kør celle 00 (opsætning, skal køres hver session).
2
Sæt CURRENT_ROUND i celle 01-A til det nye rundenummer.
3
Kopier kampene fra Cognito Forms og indsæt i raw_match_input i celle 01-C.
4
Kør celle 01-D. Henter fixture_id via football-data.org → OpenLigaDB → API-Sports.
5
Kør celle 02 for odds fra The Odds API + ELO/Opta rankings.
6
Kør celle 08 for at opdatere dashboardet og pushe til GitHub Pages.
Tilføj spillernes gæt + resultater
1
Opdater Excel-filen og kør celle 03-B med KILDE='excel' — eller indsæt Rod-tabellen i RAW_PASTE og kør med KILDE='paste'.
2
Kør celle 04-A for at hente resultater automatisk (football-data.org / OpenLigaDB / API-Sports).
3
Kør celle 05-B for at beregne rigtige gæt, H2H-dueller og stilling.
4
Kør celle 08 igen — siden opdateres automatisk på GitHub Pages inden for ~60 sekunder.
☁ H2H-systemet
Hver runde møder du én modstander.
Den der gætter flest rigtige vinder og får 3 H2H-point.
Uafgjort → 1 point til begge. Tab → 0 point.

Stillingen er baseret på akkumulerede H2H-point.
Rigtige gæt bruges som tiebreaker og vises separat.
☁ Alt i skyen
Google Colab + Google Drive giver adgang fra alle enheder.
CSV-filer gemmes i fodboldquiz/data/.
Dashboard hostes gratis på GitHub Pages.
GitHub Pages setup (én gang):
1. Opret repo på github.com (public)
2. Settings → Pages → Source: main branch
3. Sæt GITHUB_TOKEN, GITHUB_USER, GITHUB_REPO i celle 08
4. Kør celle 08 — siden er live automatisk

🔑 Nøgler du skal have:
· football-data.org (gratis, 30 sek)
· The Odds API (gratis, 500 req/md)
· GitHub token (gratis, scope: repo)
